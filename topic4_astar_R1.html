📂 How to Upload to GitHub Pages (Step-by-Step)

1

Create a GitHub account

Go to github.com → click Sign up → enter your email, password, and choose a username (e.g. suntanhk).
Verify your email address to activate the account.

2

Create a new repository

After logging in:
• Click the + icon (top right) → New repository
• Repository name: bio-flashcards
• Set to Public (required for free GitHub Pages)
• Tick ✅ Add a README file
• Click Create repository

✅ Public = free hosting. Private repos need a paid plan for Pages.
3

Upload your HTML file

On your new repository page:
• Click Add file → Upload files
• Drag and drop your index.html file (or click to browse)
• Scroll down → click Commit changes

⚠️ The file MUST be named index.html — this is the default page GitHub Pages loads.
4

Enable GitHub Pages

In your repository:
• Click Settings (top menu)
• Scroll left sidebar → click Pages
• Under Source → select Deploy from a branch
• Branch: choose main → folder: / (root)
• Click Save

5

Wait 2–3 minutes then visit your site

Your site will be live at:

https://YOUR-USERNAME.github.io/bio-flashcards

For example, if your username is suntanhk:

https://suntanhk.github.io/bio-flashcards
✅ Works on iPhone, Android, and all browsers. Completely free and permanent.
6

Updating your flashcards (when you get new files)

To replace the file with a newer version:
• Go to your repository → click on index.html
• Click the pencil ✏️ icon (or go to Add file → Upload files)
• Upload the new file → click Commit changes
• Wait 1–2 minutes → refresh your site URL

✅ You can also upload multiple HTML files (e.g. topic4.html, topic5.html) and link them from index.html.
7

Save to iPhone Home Screen (like an App)

Open your site in Safari on iPhone:
• Tap the Share button (box with arrow at bottom)
• Tap Add to Home Screen
• Tap Add

The flashcard app now appears as an icon on your home screen — just like a real app!

✅ Works offline once loaded. No App Store needed.
